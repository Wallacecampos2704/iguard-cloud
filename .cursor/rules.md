\# Regras do Projeto iGuard



Você está desenvolvendo o iGuard, um SaaS B2B de monitoramento inteligente para integradores de segurança eletrônica, TI, portaria remota, condomínios e empresas.



\## Stack

\- Front-end: Next.js, TypeScript, Tailwind CSS

\- Back-end: NestJS, TypeScript

\- Banco futuro: PostgreSQL com Prisma

\- Fila futura: Redis + BullMQ

\- Worker futuro: Node.js/NestJS

\- Pagamentos futuros: Mercado Pago

\- Automações futuras: n8n

\- Notificações futuras: E-mail, Telegram e WhatsApp



\## Regras obrigatórias

1\. Não criar código quebrado.

2\. Não apagar arquivos sem necessidade.

3\. Não executar Ping, RTSP, SIP ou TCP no navegador.

4\. Checks reais devem rodar futuramente no worker.

5\. O front-end deve ser bonito, intuitivo, premium e profissional.

6\. O dashboard deve ser entendido em 5 segundos.

7\. Toda interface deve estar em português brasileiro.

8\. Criar sistema preparado para multiempresa.

9\. Trial gratuito: 10 dias e até 3 equipamentos.

10\. Após vencer o trial, bloquear uso até pagamento aprovado.

11\. Pagamento aprovado pelo Mercado Pago deve liberar automaticamente futuramente.

12\. Preparar alertas por e-mail, Telegram e WhatsApp.

13\. Cada cliente poderá cadastrar até 5 contatos de alerta.

14\. Criar dashboard do cliente e dashboard master do dono do SaaS.

15\. Não implementar pagamento real nesta primeira etapa.

16\. Não implementar worker real nesta primeira etapa.



\## Visual

Produto premium, moderno e confiável.



Status:

\- Verde: Online

\- Amarelo: Atenção

\- Vermelho: Offline

\- Cinza: Desconhecido

\- Azul: Em manutenção



Linguagem simples:

\- Não usar: Connection timeout

\- Usar: O equipamento não respondeu no tempo esperado.
